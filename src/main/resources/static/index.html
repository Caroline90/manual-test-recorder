Chrome extension recorder

New

Load the unpacked extension from this repository, open the popup, and click Start recording. The extension records clicks plus committed text entry from inputs and text areas without showing any in-page picker, then sends each action to /api/events for XRAY-ready documentation. The popup defaults to http://localhost:8090/api/events, but you can enter a different port or full URL whenever the backend changes. Add an XRAY ticket in the popup so every exported screenshot file stays tied to the right issue.

  1. Open chrome://extensions.
  2. Enable Developer mode.
  3. Choose Load unpacked and select this repository folder.
  4. Use the popup to start recording actions from the current tab.

XRAY screenshot upload workflow

CSV + attachments

XRAY CSV import creates the manual test steps, but screenshots are usually easier to manage as issue attachments or execution evidence. The evidence bundle gives you both formats:

  • xray-steps.csv for a standard XRAY CSV import.
  • xray-steps-with-screenshots.csv with an extra Screenshot column referencing each exported image file.
  • screenshots/step-01.png by default, or screenshots/xray-123-step-01.png when you set an XRAY ticket before export.
Recommended flow
  1. Import the CSV into XRAY.
  2. Open the created Jira/XRAY test.
  3. Upload the matching files from the bundle's screenshots folder.
  4. Reference the file names from the Screenshot column in your step documentation or evidence.

  1. CSV and ZIP exports keep the same flow.
  2. Screenshot references become names like screenshots/xray-123-step-01.png.
  3. This makes multiple downloads easy to map back to the Jira/XRAY issue.
